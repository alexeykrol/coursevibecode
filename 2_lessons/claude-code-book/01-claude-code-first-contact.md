# Глава 1. Первое знакомство с Claude Code: что это такое, где он работает и как безопасно начать

**Для кого эта глава**

Эта глава написана для человека, который:
- хорошо понимает свою предметную область, но не считает себя техническим специалистом;
- хочет использовать Claude Code как рабочий инструмент, а не как игрушку;
- боится случайно что-то испортить в проекте;
- хочет понять, что именно делать в первые 30-60 минут знакомства.

Если вы никогда не работали с терминалом, настройками и рабочими папками, это нормально.
Эта глава специально написана так, чтобы можно было начать без технической подготовки.

## Зачем нужна именно такая первая глава

Почти все новички делают один и тот же ход.
Они открывают Claude Code и пишут что-то вроде:
- "разберись";
- "сделай красиво";
- "исправь все".

Потом получают слишком общий, слишком смелый или просто странный результат.
И делают вывод, что агент хаотичный.

Проблема чаще всего не в самом агенте.
Проблема в неправильном первом контакте.

Правильный первый контакт нужен для трех вещей:
1. понять, что Claude Code вообще умеет;
2. увидеть, где проходят границы безопасности;
3. получить первый полезный результат без риска повредить проект.

## Что вы сможете после этой главы

После этой главы вы сможете:
- простыми словами объяснить, что такое Claude Code;
- выбрать удобную поверхность работы;
- правильно открыть проект;
- провести первую безопасную сессию;
- задать агенту первые полезные вопросы;
- не толкнуть его слишком рано в большие изменения.

## Главная мысль главы

Claude Code — это не просто чат, который отвечает на вопросы.

Это рабочий агент, который может:
- читать ваш проект;
- объяснять его устройство;
- предлагать план;
- менять файлы;
- запускать команды;
- работать в нескольких средах;
- позже участвовать в более длинных рабочих процессах.

Именно поэтому с ним полезно обращаться не как с поисковой строкой, а как с помощником, которому нужна ясная работа и понятные границы.

## Где Claude Code вообще работает

По актуальной документации Anthropic Claude Code можно использовать в нескольких поверхностях:
- в терминале;
- в VS Code;
- в JetBrains IDE;
- в Desktop app;
- в веб-версии;
- в некоторых удаленных и командных сценариях.

Важно понять простую вещь:
это один и тот же движок, но вход в него бывает разным.

## Что выбрать нетехническому человеку на старте

Если вы совсем не технический пользователь, лучший порядок обычно такой:
1. сначала Desktop app или VS Code;
2. потом, когда станет спокойнее, терминал;
3. позже — веб, удаленные сессии и более длинные фоновые задачи.

Почему так:
- проще видеть файлы и изменения;
- меньше ощущение, что вы "залезли в системные дебри";
- легче контролировать, что именно агент собирается менять.

## Вариант 1. Desktop app

По официальной документации Desktop app — это отдельное приложение Claude, где есть вкладка `Code`.
Там можно:
- вести несколько сессий;
- смотреть visual diff;
- переключать permission modes;
- запускать задачи в облаке;
- работать с проектом без терминала.

### Как зайти впервые

1. Скачайте и установите приложение Claude.
Официальные ссылки на установку ведут с документации Claude Code и сайта `claude.ai`.
2. Запустите приложение.
3. Выполните вход в аккаунт.
Если вход еще не выполнен, приложение само попросит авторизацию.
4. После входа посмотрите в верхнюю часть окна приложения.
Там есть основные вкладки.
5. Найдите вкладку `Code`.
Если вы видите несколько вкладок и одну из них с названием `Code`, нажмите именно ее.
6. После этого откроется рабочее пространство Claude Code.

### Как понять, что вы попали туда, куда нужно

Вы должны увидеть:
- область с разговорами или сессиями;
- поле ввода сообщения;
- элементы управления режимом работы рядом с областью ввода;
- при наличии проекта — список файлов или возможность открыть проект.

Если вкладки `Code` нет:
- проверьте, что вы вошли в правильный аккаунт;
- обновите приложение до актуальной версии;
- закройте и снова откройте приложение.

## Вариант 2. VS Code

По официальной документации Anthropic расширение Claude Code ставится через Extensions view.

### Как установить

1. Откройте VS Code.
2. Посмотрите на левую вертикальную панель.
Там есть набор значков.
3. Найдите значок `Extensions`.
Обычно он выглядит как блоки или квадраты.
4. Нажмите на него.
5. В верхней части панели Extensions находится строка поиска.
6. Введите:

```text
Claude Code
```

7. Когда в списке появится расширение Claude Code, нажмите кнопку `Install`.
8. Дождитесь окончания установки.

### Как открыть Claude Code после установки

По документации есть два удобных пути.

#### Путь через Command Palette

1. Нажмите `Cmd+Shift+P` на Mac или `Ctrl+Shift+P` на Windows/Linux.
2. Откроется командная палитра.
Это длинная строка поиска вверху окна редактора.
3. Начните печатать:

```text
Claude Code
```

4. В выпадающем списке найдите команду `Open in New Tab`.
5. Нажмите на нее.
6. Должна открыться новая вкладка Claude Code внутри VS Code.

#### Путь через боковую панель

Если после установки появился значок Claude Code на боковой панели, нажмите на него.
После этого должна открыться панель с разговором и полем ввода.

Если вы не уверены, какой путь сработает, используйте палитру команд.
Для новичка это надежнее.

## Вариант 3. Терминал

Терминал — это текстовое окно команд.
Звучит страшнее, чем есть на самом деле, но для первого дня это не обязательно самый дружелюбный вход.

По документации Claude Code можно установить нативно, через Homebrew, WinGet и другие способы.
 Если вы не знаете как установить терминал или как вызвать - спросите Claude.
После установки базовый запуск выглядит так:

```bash
cd your-project
claude
```

### Что это означает человеческим языком

- `cd your-project` — перейти в папку проекта;
- `claude` — запустить Claude Code внутри этой папки.

### Как выглядит первый вход

1. Откройте Terminal на Mac или другой терминал, который вы обычно используете.
2. Введите команду перехода в нужную папку проекта.
3. Нажмите `Enter`.
4. Затем введите:

```bash
claude
```

5. Нажмите `Enter`.
6. Если это первый запуск, Claude Code попросит вас войти в аккаунт.
7. После входа появится интерактивная сессия.

Если команда `claude` не находится:
- значит Claude Code еще не установлен;
- или терминал не видит его в PATH.

Для нетехнического пользователя это нормальная точка, на которой лучше вернуться к Desktop app или VS Code, а не пытаться бороться с терминалом в первый день.

## Вариант 4. Веб

По официальной документации Claude Code можно запускать и в вебе: на `claude.ai/code`.

Это особенно полезно, когда вы хотите:
- запустить длинную задачу и вернуться позже;
- работать с репозиторием без локальной установки;
- продолжить задачу не за своим компьютером.

Для первого знакомства веб не обязателен.
Но знать о нем полезно.

## Что важно понять до первого запуска

Перед первой реальной задачей полезно понять пять вещей.

### 1. Агенту нужна правильная папка

Если вы открыли не тот проект, ответы будут странными.
Это не обязательно ошибка агента.
Очень часто это просто неправильная рабочая папка.

### 2. Первая сессия должна быть безопасной

Не просите сразу переписать весь проект.
Первый режим должен быть таким:
- читать;
- объяснять;
- показывать структуру;
- предлагать план.

### 3. Первый проект должен быть понятным вам самому

Лучше взять:
- курс;
- набор документов;
- книгу;
- рабочую папку с текстами;
- понятный небольшой проект.

Не лучший выбор для первого часа:
- огромная кодовая база;
- проект, в котором вы сами не ориентируетесь;
- важная система, где страшно ошибиться даже на один процент.

### 4. Сначала обзор, потом план, потом действие

Это правило сэкономит много нервов.

### 5. Сильный пользователь — это не тот, кто знает больше жаргона

Сильный пользователь агента умеет:
- ясно поставить задачу;
- ограничить область;
- проверить результат;
- не путать разговор и исполнение.

## Подготовка к первой сессии

### Шаг 1. Выберите один реальный проект

Это должна быть папка с настоящими материалами.
Лучше, если вы уже примерно знаете, что внутри.
Далее мы будем испоьзовать Claude с инструкциями для обработки главы книги, так что хорошо чтобы в этой папке у вас был какой-то документ. Еще лучше - просто скачайте проект этой книги и откроете его потом в Claude.
Если у вас нету своих проектов, то Вы можете:
- Взять одну из папок на вашем компьютере из примеров выше. Можете вообще взять любую и положить туда любой текстовый документ (md, txt) с разделами или главами.
- можете сделать копию проекта этой книги через git clone. Если не знаете как - спросите Claude.
- можете скачать проект как zip файл. потом разархивируйте в папку. и это и будет ваш проект. Если не знаете как - спросите Claude.
- Можете попросить Claude - помочь вам найти или сделать любой тестовый проект для изучения книги по Claude Code. и если не поймете как взять - попросите написать инструкции - например как это сделать с git clone.


### Шаг 2. Решите, что первая сессия будет без изменений

Это главное защитное правило.

### Шаг 3. Подготовьте маленькую цель

Например:
- понять структуру проекта;
- найти главные файлы;
- увидеть, где у новичка будут вопросы;
- получить короткое описание разделов.

### Шаг 4. Запишите три первых вопроса

Хороший набор такой:
1. Что это за проект?
2. Какие здесь самые важные файлы?
3. Если бы ты объяснял этот проект новичку, с чего бы начал?

## Первая безопасная сессия в Desktop app

1. Откройте приложение Claude.
2. Перейдите на вкладку `Code`.
3. Откройте нужный проект или сессию.
4. Посмотрите в нижнюю часть окна.
Там находится поле ввода сообщения.
5. Рядом с полем ввода найдите переключатель режима.
По документации Desktop app показывает mode selector рядом с кнопкой отправки.
6. Если вы видите режим `Ask permissions`, выберите его.
Это лучший стартовый режим для новичка.
7. Вставьте такой запрос:

```text
Сначала ничего не меняй.
Объясни, что это за проект, какие здесь главные файлы и с чего новичку лучше начать.
```

8. Отправьте сообщение.
9. Прочитайте ответ и проверьте: агент объясняет проект или уже пытается что-то менять.

Если он начал предлагать слишком много действий, это не катастрофа.
Просто сузьте рамку:

```text
Пока только обзор. Без изменений и без команд.
```

## Первая безопасная сессия в VS Code

1. Откройте проект в VS Code.
2. Откройте Claude Code через `Open in New Tab` или боковую панель.
3. Убедитесь, что видите поле ввода сообщения.
4. Если рядом есть элементы выбора режима, выберите осторожный режим работы.
Если режим не очевиден, просто дайте текстовое ограничение.
5. Введите тот же стартовый запрос:

```text
Сначала ничего не меняй.
Объясни, что это за проект, какие здесь главные файлы и с чего новичку лучше начать.
```

6. Отправьте сообщение.
7. Прочитайте ответ и проверьте, не вышел ли агент за рамки обзора.

## Плохой и хороший старт

### Плохой старт

```text
Разберись и исправь все, что мешает.
```

Почему плохо:
- неясна цель;
- нет материала;
- нет ограничений;
- агенту слишком рано дают широкую свободу.

### Хороший старт

```text
Пока ничего не меняй.
Объясни, что это за проект.
Покажи главные файлы.
Скажи, с чего новичку лучше начать чтение.
```

Почему хорошо:
- ясная цель;
- безопасный режим;
- нет лишней самостоятельности.

## Что нельзя делать в первый день

Не надо в первый день:
- просить переписать весь проект;
- включать самый свободный permission mode;
- подключать все расширения и интеграции подряд;
- смешивать обзор, план и исполнение в одной туманной просьбе;
- делать вид, что вы обязаны все понять мгновенно.

## Практическое упражнение

### Цель упражнения

Провести первую сессию без риска и увидеть, что Claude Code умеет читать и объяснять проект.

### Что сделать

1. Откройте один реальный проект в удобной для вас поверхности.
2. Убедитесь, что задача будет только на чтение и объяснение.
3. Вставьте этот текст:

```text
Сначала ничего не меняй.
Объясни, что это за проект, какие здесь главные файлы и какой порядок знакомства с ними ты бы предложил новичку.
```

4. Отправьте сообщение.
5. Проверьте, соответствует ли ответ запросу.

### Как понять, что упражнение выполнено хорошо

Хороший результат выглядит так:
- агент не начал ничего менять;
- он объяснил проект простыми словами;
- вы увидели структуру и следующий шаг;
- тревога от первого контакта стала меньше.

## Как эта глава связана с более глубокой логикой курса

Эта глава не про глубину настроек и не про сложную архитектуру.
Она про правильный первый контакт.

Если он сделан хорошо, дальше становится возможным:
- учиться ставить задачи;
- вводить постоянные инструкции через `CLAUDE.md` и память;
- постепенно повышать автономность;
- строить уже не хаотичное общение, а рабочую систему.

## Словарь терминов

**Claude Code**
Рабочий агент Anthropic для работы с проектами, файлами, командами и задачами разработки.

**Поверхность работы**
Конкретное место, где вы используете Claude Code: терминал, VS Code, Desktop app, веб и так далее.

**Desktop app**
Отдельное приложение Claude, где есть вкладка `Code` для работы с проектами.

**VS Code extension**
Расширение для Visual Studio Code, которое добавляет Claude Code прямо в редактор.

**Terminal**
Текстовое окно команд. Более строгая, но мощная рабочая среда.

**Permission mode**
Режим разрешений, который определяет, когда Claude должен спрашивать перед действиями.

**Ask permissions**
Осторожный режим, в котором Claude спрашивает перед изменением файлов и запуском команд. В документации он рекомендован новым пользователям.
