# Глава 2. Как ставить задачи Claude Code так, чтобы он не угадывал лишнего

**Для кого эта глава**

Эта глава написана для человека, который:
- уже открыл Claude Code и понял, что это рабочий агент, а не просто чат;
- замечает, что хорошие ответы бывают не всегда;
- хочет научиться говорить с агентом так, чтобы результат был предсказуемым;
- не хочет заучивать сложный технический язык, но хочет получать взрослый рабочий результат.

## Зачем нужна эта глава

Большинство проблем в работе с агентом начинаются не потому, что агент слабый.
Они начинаются потому, что задача была дана слишком туманно.

Например:
- "разберись";
- "улучши текст";
- "исправь все";
- "наведи порядок".

Для человека внутри головы смысл может быть ясен.
Для агента это туман.

Когда задача туманная, агент начинает достраивать недостающие детали сам.
То есть угадывать.

Наша цель в этой главе очень простая:
научиться формулировать работу так, чтобы Claude Code меньше угадывал и больше действовал в понятной рамке.

## Что вы сможете после этой главы

После этой главы вы сможете:
- отличать разговор с агентом от постановки задачи;
- использовать простой рабочий шаблон запроса;
- правильно задавать цель, материал, ограничения и признак готовности;
- понимать, когда нужен план, а когда можно сразу переходить к действию;
- снижать хаос не магией, а ясностью формулировки.

## Главная мысль главы

Агент не читает мысли.
Он работает с тем, что вы ему дали.

Хорошая задача почти всегда отвечает на четыре вопроса:
1. что именно нужно сделать;
2. на каком материале это делать;
3. какие есть ограничения;
4. по какому признаку считать работу завершенной.

Это можно записывать и в более короткой форме.

```text
Сделай вот это.
Работай только вот здесь.
Учитывай вот эти правила.
Готово, когда получится вот такой результат.
```

## Разговор и задача — не одно и то же

Это очень важная мысль.

### Режим 1. Разговор

Например:
- "Объясни, что это за проект";
- "Какие здесь главные файлы";
- "С чего новичку лучше начать".

Это режим ориентации.

### Режим 2. Задача

Например:
- "Перепиши только вступление";
- "Найди пять слабых мест";
- "Составь план и пока ничего не меняй".

Это уже рабочий контракт на один шаг.

## Простой рабочий шаблон

Вот форма, которая подходит почти всегда.

```text
Задача:
[что именно нужно сделать]

Материал:
[какие файлы, папки, документы или фрагменты относятся к задаче]

Ограничения:
[что нельзя менять, какой стиль держать, насколько глубоко можно заходить]

Готово, когда:
[по какому признаку я пойму, что задача выполнена правильно]
```

Чем выше риск ошибки, тем важнее не пропускать ни один из этих блоков.

## Четыре части хорошей задачи

### 1. Что именно нужно сделать

Это должно быть одно понятное действие.

Плохо:
- "улучши курс";
- "сделай все понятнее".

Лучше:
- "Перепиши вступление более простым языком";
- "Найди три места, где новичок запутается";
- "Составь план улучшения главы без редактирования файла".

### 2. На каком материале работать

Агенту нужен не только смысл, но и место.

Указывайте:
- файл;
- папку;
- раздел;
- фрагмент;
- конкретную ошибку.

Например:

```text
Работай только с файлом 2_lessons/claude-code-book/01-claude-code-first-contact.md.
```

Если вы работаете в VS Code, можно указывать путь текстом прямо в сообщении.
Если интерфейс поддерживает добавление контекста через выделение или @-упоминание файла, это тоже можно использовать.
Но текстовый путь для новичка надежнее.

### 3. Какие есть ограничения

Это то, что удерживает агента от лишней самостоятельности.

Примеры:
- "Не меняй другие файлы";
- "Не меняй смысл, только упрости язык";
- "Сначала дай план";
- "Не используй технический жаргон";
- "Пока ничего не меняй, только анализ".

### 4. Как мы поймем, что задача выполнена

Это самая недооцененная часть.

Примеры:
- "Готово, когда я увижу новый вариант вступления объемом до 250 слов";
- "Готово, когда ты перечислишь пять проблемных мест и объяснишь каждое";
- "Готово, когда ты покажешь план, но не начнешь редактировать файл".

## Плохие и хорошие формулировки

### Пара 1

Плохо:

```text
Сделай эту главу лучше.
```

Почему плохо:
- непонятно, какую именно главу;
- непонятно, что значит "лучше";
- непонятно, можно ли менять структуру.

Лучше:

```text
Перепиши вступление в файле 2_lessons/claude-code-book/01-claude-code-first-contact.md более простым языком.
Не меняй остальную главу.
Готово, когда вступление останется коротким и понятным человеку без технического опыта.
```

### Пара 2

Плохо:

```text
Наведи тут порядок.
```

Лучше:

```text
Посмотри файл 2_lessons/claude-code-book/README.md и предложи новую структуру оглавления.
Пока ничего не меняй.
Готово, когда ты покажешь 5-7 пунктов нового оглавления и объяснишь логику.
```

### Пара 3

Плохо:

```text
Разберись и исправь.
```

Лучше:

```text
Прочитай файл 2_lessons/claude-code-book/01-claude-code-first-contact.md и найди места, где новичок может потеряться в интерфейсных действиях.
Пока ничего не меняй.
Готово, когда ты покажешь список таких мест и предложишь, что надо разъяснить подробнее.
```

## Когда нужен план

По документации Claude Code есть `Plan mode`.
Он особенно полезен, когда задача:
- большая;
- размытая;
- рискованная;
- состоит из нескольких шагов.

### План нужен, если:
- вы пока не уверены в направлении;
- боитесь, что агент начнет менять лишнее;
- задача многослойная;
- надо сначала согласовать путь.

### План можно не просить, если:
- задача очень маленькая;
- вы указали один файл;
- границы и результат очевидны;
- изменение легко проверяется.

## Как включить режим "сначала план"

### В Desktop app

1. Откройте сессию Claude Code.
2. Посмотрите в нижнюю часть окна, где находится поле ввода сообщения.
3. Рядом с ним найдите selector режима.
4. Если вы видите вариант `Plan mode`, выберите его.
5. После этого введите задачу.
6. Агент должен сначала анализировать и строить план, а не менять файлы.

### В VS Code

1. Откройте вкладку или панель Claude Code.
2. Найдите нижнее поле ввода.
3. Если рядом есть переключатель режима, выберите `Plan mode`.
4. Затем отправьте задачу.

### Если вы не видите переключатель режима

Просто напишите текстом:

```text
Сначала составь план. Пока ничего не меняй.
```

Это не менее полезно.

## Очень хороший прием: сначала взять у вас интервью

Если у вас есть смутная идея, не пытайтесь сразу придумать идеальный запрос.

Можно написать так:

```text
У меня пока сырая идея. Сначала задай мне 5-7 уточняющих вопросов, чтобы превратить ее в ясную рабочую задачу. Пока ничего не меняй.
```

Это один из самых практичных способов превратить туман в нормальную работу.

## Самая безопасная схема для новичка

Если вы не уверены, как формулировать задачи, используйте такую лестницу.

### Шаг 1. Попросите обзор

```text
Объясни, что здесь происходит.
Пока ничего не меняй.
```

### Шаг 2. Попросите выделить проблему

```text
Покажи, где здесь самые слабые места для новичка.
Пока только анализ.
```

### Шаг 3. Попросите план

```text
Предложи план исправления этих слабых мест.
Пока не редактируй файлы.
```

### Шаг 4. Дайте одну маленькую задачу

```text
Сделай только первый шаг из этого плана.
Работай только в одном файле.
```

## Частые ошибки

### Ошибка 1. Давать сразу слишком большую задачу

```text
Перестрой мне весь курс.
```

### Ошибка 2. Не указывать материал

```text
Сделай объяснение проще.
```

Какое объяснение? В каком файле? В каком разделе?

### Ошибка 3. Не ставить границы

```text
Исправь все, что считаешь нужным.
```

### Ошибка 4. Смешивать много задач в одну кучу

```text
Объясни структуру, перепиши вступление, обнови оглавление и предложи новый план курса.
```

### Ошибка 5. Не говорить, что пока нельзя менять файлы

Простая фраза:

```text
Пока ничего не меняй.
```

Очень полезна.

## Практическое упражнение

### Цель упражнения

Попросить Claude Code проанализировать уже готовую первую главу, но не менять ее.

### Что сделать

1. Откройте Claude Code в проекте книги.
2. Нажмите в поле ввода сообщения.
3. Вставьте этот текст:

```text
Задача:
Найди места, где новичок может потеряться в главе.

Материал:
Работай только с файлом 2_lessons/claude-code-book/01-claude-code-first-contact.md.

Ограничения:
Пока ничего не меняй. Сначала только анализ. Смотри особенно на места, где человеку нужно что-то нажать, открыть или настроить.

Готово, когда:
Ты покажешь 5 конкретных проблемных мест и для каждого объяснишь, что именно надо разъяснить подробнее.
```

4. Отправьте сообщение.
5. Проверьте, выполнил ли агент все четыре части задачи.

## Как эта глава связана с более глубокой логикой курса

На этом этапе вы еще не строите сложную систему.
Но вы уже делаете первый взрослый шаг:
переводите общение с агентом из тумана в ясную рабочую рамку.

Именно отсюда потом вырастают:
- постоянные инструкции проекта;
- устойчивые настройки;
- repeatable workflows;
- безопасная автономность;
- `Agent Operating System`.

## Словарь терминов

**Постановка задачи**
Способ дать агенту не просто тему разговора, а четкое рабочее задание.

**Контекст**
Материал, на котором агент должен работать: файлы, папки, документы, примеры.

**Ограничения**
Правила и запреты, которые удерживают агента от лишних действий.

**Критерий готовности**
Признак, по которому можно понять, что задача выполнена правильно.

**Plan mode**
Режим, в котором Claude Code сначала строит план и не спешит сразу менять файлы.
