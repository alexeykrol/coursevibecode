# Прикладные Playbooks

Это раздел с готовыми рабочими сценариями.

Если книги по Codex и Claude Code объясняют принципы, то playbooks отвечают на другой вопрос:
что именно делать в реальной ситуации шаг за шагом.

## Для кого этот раздел

Этот раздел нужен человеку, который:
- уже прочитал хотя бы первые главы одной из книг;
- не хочет каждый раз придумывать процесс заново;
- хочет брать готовый сценарий и повторять его без лишнего хаоса.

## Как устроены playbooks

Каждый playbook — это один повторяемый сценарий.
Внутри обычно есть:
- когда этот сценарий нужен;
- как подготовиться;
- пошаговая работа;
- отдельные ветки для Codex и Claude Code;
- хорошие и плохие формулировки;
- частые ошибки;
- короткое упражнение;
- словарь терминов.

1. [01-how-to-choose-between-codex-and-claude-code.md](./01-how-to-choose-between-codex-and-claude-code.md) — как понять, с какого агента вам лучше начать.
2. [02-first-safe-project-review.md](./02-first-safe-project-review.md) — как безопасно попросить агента объяснить новый проект без изменений файлов.
3. [03-first-safe-edit.md](./03-first-safe-edit.md) — как провести первую маленькую правку без лишнего риска.
4. [04-turn-outline-into-full-chapter.md](./04-turn-outline-into-full-chapter.md) — как превратить конспект в полноценную главу с помощью агента.
5. [05-regular-editorial-pass.md](./05-regular-editorial-pass.md) — как делать регулярный редакторский проход по уже написанным материалам.
6. [06-weekly-content-production-cycle.md](./06-weekly-content-production-cycle.md) — как выстроить недельный цикл производства контента с агентом.
7. [07-building-project-instructions.md](./07-building-project-instructions.md) — как собрать проектные инструкции, чтобы агент не забывал правила работы.
8. [08-safe-growth-of-autonomy.md](./08-safe-growth-of-autonomy.md) — как безопасно повышать автономность агента без потери контроля.
9. [09-agent-organization-for-small-team.md](./09-agent-organization-for-small-team.md) — как организовать работу маленькой команды с агентами без лишнего хаоса.

## Как читать этот раздел

Если вы новичок, идите так:
1. сначала [01-how-to-choose-between-codex-and-claude-code.md](./01-how-to-choose-between-codex-and-claude-code.md);
2. потом [02-first-safe-project-review.md](./02-first-safe-project-review.md);
3. затем [03-first-safe-edit.md](./03-first-safe-edit.md);
4. после этого можно переходить к более содержательным сценариям вроде [04-turn-outline-into-full-chapter.md](./04-turn-outline-into-full-chapter.md) и [05-regular-editorial-pass.md](./05-regular-editorial-pass.md).
5. когда эти базовые сценарии уже привычны, переходите к [06-weekly-content-production-cycle.md](./06-weekly-content-production-cycle.md), чтобы собрать устойчивый недельный ритм.
6. после этого логично открыть [07-building-project-instructions.md](./07-building-project-instructions.md), чтобы закрепить рабочие правила проекта во внешнем файле, а не только в памяти текущей сессии.
7. затем переходите к [08-safe-growth-of-autonomy.md](./08-safe-growth-of-autonomy.md), когда захотите расширять самостоятельность агента без скачка в хаос.
8. после этого открывайте [09-agent-organization-for-small-team.md](./09-agent-organization-for-small-team.md), если работа уже выходит за пределы одного исполнителя и нужен минимальный рабочий контур для маленькой команды.

## Связь с книгами

Если вы хотите сначала понять основы, а потом идти по сценариям, используйте такой маршрут:
- [Книга по Codex](../codex-book/README.md)
- [Книга по Claude Code](../claude-code-book/README.md)

Playbooks не заменяют книги.
Они превращают знания из книг в повторяемые практические действия: от первых безопасных шагов до недельных циклов, проектных инструкций, управляемой автономности и минимальной агентной организации.
