# Playbook 5. Как делать регулярный редакторский проход по уже написанным материалам

**Для кого этот playbook**

Этот сценарий нужен человеку, который:
- уже накопил несколько глав, уроков или документов;
- замечает, что материалы начинают расползаться по стилю;
- хочет не просто править случайные фразы, а делать системный редакторский проход;
- хочет превратить это в повторяемый цикл.

## Когда использовать этот сценарий

Используйте его:
- после написания нескольких глав подряд;
- перед публикацией большой части курса;
- когда чувствуете, что структура уже есть, но качество не выровнено.

## Какой результат вы должны получить

В конце сценария у вас должно быть:
- список конкретных проблем по текстам;
- единый редакторский фокус на один проход;
- исправленные материалы без лишней полной переписи;
- короткое резюме, что стало лучше.

## Главная мысль

Редакторский проход полезен тогда, когда он фокусный.

Если в один заход пытаться одновременно:
- переписать стиль;
- улучшить логику;
- поправить терминологию;
- выровнять структуру;
- проверить факты;

то вы быстро теряете управляемость.

Лучше делать проходы по одной главной оси.

## Пять возможных осей редакторского прохода

1. Ясность языка.
2. Единообразие терминов.
3. Интерфейсная конкретика.
4. Связность между главами.
5. Практическая плотность.

## Самый полезный первый редакторский проход

Для нетехнического курса особенно полезен такой первый фокус:

```text
Найди места, где новичок остановится,
потому что ему не объяснили буквально,
что и как делать.
```

Это сразу повышает практическую ценность материала.

## Подготовка

Перед проходом выберите:
1. какие файлы входят в проход;
2. одну ось проверки;
3. формат результата от агента.

Очень удобный формат результата:
- 5-10 конкретных проблем;
- почему это проблема;
- что именно лучше исправить.

## Ветка A. Если вы работаете в Codex

### Подходящий запрос

```text
Сделай редакторский проход по этим файлам.
Фокус только один: места, где новичок может остановиться из-за недостатка интерфейсной конкретики.
Не переписывай файлы целиком.
Сначала дай список проблем:
- файл;
- проблема;
- почему это мешает;
- что лучше сделать.
Потом отдельно исправим только выбранные места.
```

### Почему это хороший порядок

Сначала вы получаете карту проблем,
а не сразу поток изменений.
Это намного управляемее.

## Ветка B. Если вы работаете в Claude Code

### Подходящий запрос

```text
Нужен редакторский проход по этим файлам.
Фокус только один: где новичок споткнется, потому что шаги описаны слишком абстрактно.
Сначала не переписывай тексты.
Сначала дай список конкретных проблем по схеме:
- файл;
- место;
- почему это проблема;
- что улучшить.
```

### Почему это хороший порядок

Он удерживает Claude Code от слишком ранней полной переработки,
когда вам сначала нужен диагноз.

## Шаг 1. Получите карту проблем

Не начинайте с тотальной переписи.
Сначала нужен диагноз.

## Шаг 2. Выберите только часть проблем на текущий проход

Например:
- только 5 самых важных;
- только одна глава;
- только одна группа ошибок.

Это сохраняет управляемость.

## Шаг 3. Исправляйте пакетами малого размера

Не нужно просить:
"исправь весь курс".

Лучше так:

```text
Исправь только эти 3 места.
Сохрани смысл.
Не переписывай остальные части глав.
```

## Шаг 4. После исправлений запросите короткий итог

Полезный запрос:

```text
Коротко подведи итог:
- что было самым частым типом проблемы;
- что мы улучшили;
- что стоит проверить на следующем проходе.
```

Так вы превращаете разовую редактуру в повторяемый рабочий цикл.

## Хорошие и плохие формулировки

### Плохо

```text
Пройди по этим текстам и сделай их лучше.
```

### Лучше

```text
Сделай редакторский проход по этим главам.
Фокус только один: места, где новичку не хватает буквальной пошаговой конкретики.
Сначала дай список проблем, потом будем править выбранные места.
```

## Частые ошибки

### Ошибка 1. Делать слишком широкий фокус

Тогда проход расползается.

### Ошибка 2. Сразу запускать полную перепись

Так вы теряете контроль над причиной изменений.

### Ошибка 3. Не разделять диагностику и исправление

Это одна из самых частых причин хаоса.

### Ошибка 4. Не фиксировать, что именно было улучшено

Тогда следующий проход опять начинается почти с нуля.

## Практическое упражнение

### Что сделать

1. Возьмите 2-3 уже написанные главы.
2. Выберите один фокус: например, интерфейсная конкретика.
3. Попросите агента дать только карту проблем.
4. Выберите 3-5 исправлений.
5. Попросите внести только их.
6. В конце попросите короткое резюме прохода.

## Словарь терминов

**Редакторский проход**
Системная проверка набора материалов по одному выбранному фокусу.

**Фокус прохода**
Главная ось проверки в текущем цикле.

**Диагностика**
Этап, где агент сначала находит и объясняет проблемы.

**Исправление**
Этап, где агент меняет только выбранные места.

**Повторяемый рабочий цикл**
Повторяемый рабочий цикл, который можно использовать снова и снова.
