# Playbook 1. Как выбрать между Codex и Claude Code без лишней путаницы

**Для кого этот playbook**

Этот сценарий нужен человеку, который:
- видит сразу два агента и не понимает, с кого начинать;
- боится выбрать "не тот" инструмент;
- не хочет изучать две среды одновременно с первого дня;
- хочет быстро принять спокойное рабочее решение.

## Когда использовать этот сценарий

Используйте его, если у вас еще нет устойчивой привычки работы ни с одним агентом.

Не используйте его, если:
- вы уже уверенно работаете с одним агентом и просто хотите второй как дополнительный;
- вам нужен глубокий технический benchmark;
- вы выбираете инструмент для большой команды с уже готовой внутренней инфраструктурой.

## Какой результат вы должны получить

В конце этого сценария у вас должно быть:
- решение, с какого агента начинать именно вам;
- понимание, почему выбран именно он;
- запасной план, когда стоит подключать второй инструмент.

## Главная мысль

Новичку не нужен "идеальный агент на все случаи жизни".
Новичку нужен такой первый агент, с которым он быстрее войдет в рабочий ритм и меньше испугается интерфейса.

Поэтому на старте нужно смотреть не на абстрактные споры в интернете, а на четыре практических вопроса:
1. где вам психологически легче работать;
2. какой интерфейс вам понятнее;
3. какая логика проекта вам ближе;
4. какой тип задач вы будете делать чаще всего в ближайший месяц.

## Короткое различие простыми словами

### Codex

Codex особенно хорошо ложится на логику:
- проект как рабочая папка;
- четкие проектные инструкции;
- последовательная работа через задачи, файлы и проверки;
- взрослая инженерная среда вокруг OpenAI.

Если сказать очень просто:
Codex ощущается как строгий и организованный рабочий кабинет.

### Claude Code

Claude Code особенно хорошо ложится на логику:
- ясное общение внутри рабочей среды;
- сильная память проекта через `CLAUDE.md` и auto memory;
- удобная линия Desktop app, VS Code и agent teams;
- очень естественный переход от одного агента к нескольким ролям.

Если сказать очень просто:
Claude Code ощущается как рабочий помощник, с которым легче наращивать совместную повседневную работу.

## Самое простое правило выбора

Если вы вообще не уверены, используйте такое правило:
- если вас тянет в экосистему OpenAI и вам нравится строгая инженерная логика, начинайте с Codex;
- если вам ближе экосистема Anthropic и идея памяти проекта через `CLAUDE.md`, начинайте с Claude Code.

Но это только грубое правило.
Ниже более точная схема.

## Схема выбора в пять вопросов

### Вопрос 1. Где вам спокойнее находиться каждый день

Если вам спокойнее:
- в среде OpenAI и ее документации — склоняйтесь к Codex;
- в среде Claude, Desktop app и логике `CLAUDE.md` — склоняйтесь к Claude Code.

### Вопрос 2. Что вас пугает больше всего

Если вас больше пугает:
- потеря структуры и хаос в проекте — начните с Codex;
- потеря контекста и необходимость все время заново объяснять проект — начните с Claude Code.

### Вопрос 3. Какой интерфейс вам ближе

Если вам удобнее:
- мыслить через проект, папки, режимы, разрешения и инженерную дисциплину — Codex;
- мыслить через постоянные инструкции, память, Desktop app, VS Code и совместную работу — Claude Code.

### Вопрос 4. Какой ближайший тип задач у вас будет доминировать

Если у вас ближайший месяц больше:
- проектной навигации, настройки рабочих правил, безопасных правок и строгого цикла проверки — Codex;
- длинной совместной работы над текстами, накопления контекста и повторяющихся редакторских циклов — Claude Code.

### Вопрос 5. Что вы хотите получить через 2-3 недели

Если вы хотите:
- более инженерную дисциплину вокруг задач и проверок — Codex;
- более гладкую ежедневную совместную работу с памятью проекта — Claude Code.

## Самая практичная стратегия для новичка

Вместо того чтобы выбирать навсегда, делайте так:
1. выберите один основной агент на 2 недели;
2. не прыгайте туда-сюда каждый день;
3. на этом агенте пройдите три сценария: обзор проекта, маленькая правка, редакторский проход;
4. только после этого пробуйте второй агент на тех же трех сценариях.

Тогда сравнение будет честным.

## Мини-тест выбора

Отвечайте быстро, не философствуя.

### Если у вас чаще такие мысли:

- "Мне нужен порядок, рамка и строгий процесс"
- "Я хочу четко видеть, что агент собирается делать"
- "Мне важно не разжимать свободу раньше времени"

То ваш первый кандидат — Codex.

### Если у вас чаще такие мысли:

- "Мне нужен помощник, который хорошо держит длинный контекст"
- "Я хочу сильнее опираться на память проекта"
- "Мне важно, чтобы ежедневная совместная работа была очень естественной"

То ваш первый кандидат — Claude Code.

## Плохие и хорошие решения

### Плохое решение

"Я попробую оба сразу, буду постоянно переключаться и потом как-нибудь пойму".

Почему это плохо:
- нет устойчивой привычки;
- все смешивается;
- вы не понимаете, что дало результат: агент или случайность.

### Хорошее решение

"Я беру один агент на две недели. На нем я прохожу три одинаковых сценария. Потом сравниваю со вторым агентом на тех же сценариях".

Почему это хорошо:
- сравнение становится честным;
- вы учитесь процессу, а не только интерфейсу;
- хаоса меньше.

## Частые ошибки

### Ошибка 1. Искать "лучшего вообще"

Практически полезнее искать не лучшего вообще, а лучшего для вашего текущего этапа.

### Ошибка 2. Перескакивать между агентами каждый день

Так вы не строите навык ни в одной среде.

### Ошибка 3. Выбирать по чужим спорам

Чужой технический контекст может не иметь к вам отношения.

### Ошибка 4. Начинать сразу с глубокой автоматизации

Сначала нужен базовый рабочий ритм.

## Практическое упражнение

### Что сделать

1. Возьмите лист бумаги или заметку.
2. Напишите два заголовка: `Codex` и `Claude Code`.
3. Под каждым напишите по три причины, почему этот вариант вам может подойти.
4. Затем ответьте на вопрос: "В какой среде я хочу спокойно провести ближайшие 2 недели?"
5. Выберите один основной агент.
6. Запишите рядом запасное правило:

```text
Через 2 недели я проверю второй инструмент на тех же трех сценариях:
1. обзор проекта;
2. маленькая безопасная правка;
3. редакторский проход.
```

## Словарь терминов

**Экосистема**
Группа связанных инструментов, документации и привычек вокруг одной платформы.

**Рабочая среда**
То место и тот порядок работы, в котором вы проводите большинство сессий с агентом.

**Память проекта**
Постоянный контекст и правила, которые помогают агенту лучше понимать именно ваш проект.

**Auto memory**
Автоматически накапливаемые заметки в Claude Code.

**`CLAUDE.md`**
Файл с постоянными инструкциями для Claude Code.

**`AGENTS.md`**
Файл с постоянными инструкциями для Codex.
