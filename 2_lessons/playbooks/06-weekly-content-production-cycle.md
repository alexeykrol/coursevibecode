# Playbook 6. Как выстроить недельный цикл производства контента с агентом

**Для кого этот playbook**

Этот сценарий нужен человеку, который:
- уже пишет статьи, уроки, главы или методички;
- хочет перестать работать рывками "когда будет настроение";
- хочет получать заметный результат каждую неделю;
- хочет использовать агента не как случайного собеседника, а как часть рабочего ритма.

## Когда использовать этот сценарий

Используйте его:
- когда у вас постоянно копятся идеи, но мало готовых материалов;
- когда каждая новая глава начинается почти с нуля;
- когда вы хотите превратить хаотичное письмо в устойчивый недельный цикл;
- когда вам нужно делать контент регулярно, а не эпизодически.

## Какой результат вы должны получить

В конце сценария у вас должно быть:
- понятное недельное устройство работы;
- один главный результат недели;
- список вторичных задач, которые поддерживают этот результат;
- короткое закрытие недели, чтобы следующая неделя не стартовала из тумана.

## Главная мысль

Недельный цикл нужен не для красоты.
Он нужен, чтобы агент помогал вам двигаться вперед предсказуемо.

Если каждая неделя выглядит так:
- в понедельник вы не знаете, за что браться;
- в среду у вас десять незакрытых кусков;
- в пятницу вы не понимаете, что именно завершено;

то проблема не в слабом агенте.
Проблема в отсутствии рабочего ритма.

Агент особенно полезен тогда, когда у него есть:
- ясная цель недели;
- понятный главный артефакт;
- ограниченный объем работы;
- отдельный момент для проверки и фиксации.

Важно:
недельный цикл не заменяет базовый рабочий цикл из книг.
Он стоит уровнем выше.

Внутри каждого дня у вас все равно остается тот же полезный порядок:

```text
Обзор -> План -> Действие -> Проверка -> Фиксация
```

Если этот малый цикл не работает,
недельный ритм тоже быстро станет хрупким.

## Что не надо делать

Не превращайте неделю в список из двадцати обещаний.

Для нетехнического автора лучше работает такой принцип:
- одна главная единица результата;
- две-три поддерживающие задачи;
- одно обязательное закрытие недели.

## Самая практичная недельная схема

Для большинства авторских проектов хорошо работает такой ритм:

```text
Понедельник: выбрать главный результат недели
Вторник: собрать каркас и материалы
Среда: сделать основной черновик
Четверг: провести фокусный проход
Пятница: зафиксировать итог и обновить индексные файлы
```

Это не жесткий календарь.
Это простая логика движения.

## Подготовка

Перед началом недели подготовьте:
1. папку проекта;
2. `BACKLOG.md` или другой список ближайших тем;
3. файл, который будет главным результатом недели;
4. правило, что неделя не считается завершенной без закрытия недели.

Если вы работаете в существующем репозитории, сделайте буквально следующее:
1. откройте папку проекта;
2. откройте `README.md` или локальный индекс раздела;
3. откройте `BACKLOG.md`, если он есть;
4. найдите ближайший незавершенный файл;
5. только после этого запускайте агента.

Это и есть ваш простой недельный `preflight`:
короткая проверка, что вы стартуете не из тумана, а из правильной папки и правильной очереди задач.

## Шаг 1. Выберите один главный результат недели

Плохо:

```text
На этой неделе надо сильно продвинуть весь курс.
```

Лучше:

```text
Главный результат недели:
написать одну полноценную главу 06
и обновить все связанные индексы.
```

Главный результат должен быть:
- видимым;
- проверяемым;
- конечным;
- помещаться в одну неделю.

## Шаг 2. Зафиксируйте поддерживающие задачи

Поддерживающие задачи нужны, но их должно быть мало.

Например:
- подготовить каркас главы;
- сделать один редакторский проход;
- обновить оглавление;
- обновить `SESSION-HANDOFF.md` для следующей сессии.

Если поддерживающих задач слишком много, неделя снова расползется.

## Шаг 3. Попросите агента собрать план недели

На этом этапе агент не должен сразу писать все подряд.
Сначала он должен помочь вам собрать управляемый недельный план.

Хороший запрос:

```text
Ниже текущий `BACKLOG.md` и ближайший файл.
Собери реалистичный недельный цикл.
Нужны:
- один главный результат недели;
- 2-3 поддерживающие задачи;
- порядок работы по дням;
- критерий, по которому неделя считается завершенной.
Не расширяй объем работы.
```

## Ветка A. Если вы работаете в Codex

### Как буквально начать неделю

1. Откройте нужную папку проекта в Codex.
2. Дождитесь, пока откроется рабочая сессия.
3. Вставьте контекст: ближайшие задачи из `BACKLOG.md`, локальный `README.md`, путь к главному файлу недели.
4. Дайте агенту только задачу на план недели, не на полную запись всего блока сразу.

### Подходящий запрос

```text
Работаем недельным циклом.
Вот текущий `BACKLOG.md` и ближайший файл.
Помоги собрать реалистичный план недели.
Нужны:
- один главный результат;
- 2-3 поддерживающие задачи;
- порядок по дням;
- отдельное закрытие недели в конце.
Пиши просто и практично.
Не добавляй новые большие темы сверх уже выбранного файла.
```

### Что важно проверить

Проверьте, что Codex:
- не раздул неделю до слишком большого объема;
- не предложил сразу три главных результата;
- оставил отдельное место для проверки и фиксации.

## Ветка B. Если вы работаете в Claude Code

### Как буквально начать неделю

1. Откройте проект в Claude Code.
2. Убедитесь, что открыт именно тот раздел, где лежит ближайшая задача.
3. Подайте агенту `BACKLOG.md`, локальный индекс и текущую точку продолжения.
4. Сначала получите план недели, а не полный поток правок.

### Подходящий запрос

```text
Нужен недельный цикл работы по текущему разделу.
Вот `BACKLOG.md` и ближайший незавершенный файл.
Собери реалистичный недельный план:
- один главный результат;
- 2-3 поддерживающие задачи;
- порядок по дням;
- что должно попасть в закрытие недели.
Не расползайся на весь проект.
```

### Что важно проверить

Проверьте, что Claude Code:
- удержал фокус на одной главной единице результата;
- не превратил неделю в абстрактный список пожеланий;
- оставил в плане реальный момент проверки качества.

## Шаг 4. Разделите неделю на пять разных режимов работы

Очень полезно не смешивать все в один день.

### Понедельник. Выбор и рамка

Здесь вы:
- выбираете главный результат;
- фиксируете, что не входит в неделю;
- просите агента собрать план.

### Вторник. Каркас и сбор материала

Здесь вы:
- собираете структуру;
- поднимаете нужные файлы;
- просите агента найти слабые места до написания основного текста.

### Среда. Основной производственный проход

Это день, когда появляется основной объем:
- черновик главы;
- новая инструкция;
- переработанный раздел;
- собранный сценарий.

### Четверг. Фокусный проход

Здесь не надо делать новую большую работу.
Здесь надо сделать один улучшительный проход.

Например:
- упростить язык;
- добавить буквальные интерфейсные шаги;
- убрать логические дыры;
- выровнять структуру.

### Пятница. Закрытие недели

Это обязательный шаг.
При закрытии недели вы:
- проверяете, что главный результат действительно завершен;
- обновляете `README.md` и индексные файлы;
- фиксируете, что осталось на следующую неделю;
- просите агента коротко описать итог недели.

## Шаг 5. Не смешивайте производство и архитектурные развилки

Если в середине недели всплывает крупный вопрос вроде:
- "надо ли перестроить весь раздел?";
- "может быть, переделаем всю структуру курса?";
- "может быть, теперь лучше писать по другой методике?";

не смешивайте это с недельным производственным циклом.

Такие вопросы выносятся отдельно.
Иначе неделя, которая должна была закончиться одной готовой главой, закончится туманным обсуждением всего проекта.

## Шаг 6. Делайте закрытие недели в явном виде

Очень полезно просить у агента короткий итог недели в одном и том же формате.

Например:

```text
Сделай итог недели.
Нужны 4 блока:
- что завершено;
- что обновлено в индексах и служебных файлах;
- что осталось незавершенным;
- с какого файла логично начать следующую неделю.
```

Именно закрытие недели превращает ее в повторяемый рабочий цикл.

## Хорошие и плохие формулировки

### Плохо

```text
Помоги на этой неделе хорошо продвинуть проект.
```

### Лучше

```text
На этой неделе главный результат один:
написать playbook 06 и обновить связанные индексы.
Помоги собрать недельный цикл:
- рамка;
- шаги по дням;
- один фокусный проход;
- закрытие недели.
Не добавляй лишние большие задачи.
```

## Частые ошибки

### Ошибка 1. Делать больше одного главного результата

Это почти всегда ломает неделю.

### Ошибка 2. Не отделять написание от редакторского прохода

Тогда вы одновременно и пишете, и сомневаетесь, и переписываете.

### Ошибка 3. Не обновлять индексы и служебные файлы

Тогда следующая сессия начинается с потери контекста.

### Ошибка 4. Не делать закрытие недели

Без закрытия недели неделя формально кончилась,
а операционно не закрылась.

### Ошибка 5. Пытаться автоматизировать неделю целиком слишком рано

Сначала надо один-два раза пройти цикл вручную.
Сильная автоматизация начинается не с расписания,
а с устойчивого ручного цикла.

## Практическое упражнение

### Что сделать

1. Выберите один главный результат на ближайшие 5 рабочих дней.
2. Запишите, что точно не входит в эту неделю.
3. Попросите агента собрать недельный цикл по дням.
4. В середине недели сделайте только один фокусный проход, а не пять разных.
5. В конце недели проведите явное закрытие недели.
6. Зафиксируйте, с чего начнется следующая неделя.

## Словарь терминов

**Недельный цикл**
Повторяемый порядок работы на одну неделю.

**Главный результат недели**
Один основной артефакт, ради которого строится неделя.

**Поддерживающие задачи**
Небольшие задачи, которые помогают довести главный результат до завершения.

**Закрытие недели**
Финальное закрытие цикла с фиксацией результата и следующей точки входа.

**Фокусный проход**
Отдельный улучшительный проход по одному выбранному критерию.
