# Урок 4: Твой первый проект за 10 минут — в чате

**Длительность:** 10-12 минут  
**Формат:** Скринкаст + voice-over  
**Ключевой месседж:** Ты создашь первое работающее приложение прямо в чате за 10 минут — и увидишь, где начинается потолок

---

## СТРУКТУРА ВИДЕО

### [00:00 - 00:20] ХУК (Скринкаст)

**ВИЗУАЛ:** Пустое окно ChatGPT, курсор мигает

**ТЕКСТ:**

10 минут. Один промпт. Работающее приложение.

Сейчас ты увидишь, как создать калькулятор, landing page или простую игру без единой строчки кода.

Но самое важное — ты увидишь границу. Где чат перестает работать. И почему тебе нужен следующий уровень.

Таймер пошел.

---

### [00:20 - 01:00] ОБЕЩАНИЕ (Voice-over + скринкаст)

**ВИЗУАЛ:** Начинаем печатать промпт

**ТЕКСТ:**

Теория закончена. Начинается практика.

За следующие 10 минут ты получишь:

**Первое:** Рабочее приложение, созданное в чате. Ты увидишь, как это просто.

**Второе:** Понимание ограничений чата. Где он работает, где ломается.

**Третье:** Чек-лист "3 сигнала, что пора выходить из чата". Ты узнаешь, когда переходить дальше.

Поехали. Создаем.

---

### [01:00 - 08:00] СОЗДАНИЕ ПРОЕКТА В ЧАТЕ (Скринкаст + комментарии)

**ВИЗУАЛ:** Полный скринкаст работы в ChatGPT

**ТЕКСТ:**

**[01:00 - 01:30] Выбор проекта**

Создадим простой калькулятор расходов. Базовый, но рабочий.

Вот промпт:

*"Создай HTML-страницу с калькулятором ежемесячных расходов. Нужно:
- Поля ввода для разных категорий (еда, транспорт, жилье, развлечения)
- Автоматический подсчет суммы
- Красивый дизайн с использованием Tailwind CSS
- Адаптивная верстка для мобильных
- Все в одном HTML-файле"*

Отправляем.

**[01:30 - 03:00] Получение кода**

Чат генерирует код. Занимает 20-30 секунд.

Получили HTML-файл. Копируем весь код.

Создаем файл на компьютере: expense-calculator.html

Вставляем код. Сохраняем.

**[03:00 - 04:00] Запуск**

Открываем файл в браузере. Двойной клик.

Работает! Приложение запустилось.

Проверяем:
- Вводим цифры в поля
- Сумма считается автоматически
- Дизайн выглядит нормально
- На телефоне тоже работает

**Время:** 4 минуты. Приложение готово.

**[04:00 - 05:30] Добавление функции**

Хотим добавить процент сбережений.

Новый промпт:

*"Добавь поле 'Доход', и автоматически считай процент сбережений: (Доход - Расходы) / Доход × 100. Покажи результат с цветом: зеленый если > 20%, желтый если 10-20%, красный если < 10%"*

Чат добавляет функцию. Копируем новый код.

Заменяем в файле. Сохраняем. Обновляем браузер.

Работает! Теперь показывает процент сбережений с цветом.

**[05:30 - 06:30] Улучшение дизайна**

Хотим круче.

*"Сделай дизайн в стиле минимализма: белый фон, большие цифры, анимация при вводе"*

Чат обновляет дизайн.

Результат: стало красивее, профессиональнее.

**[06:30 - 08:00] Попытка усложнить — вот где начинаются проблемы**

Теперь хотим сохранять данные между сессиями.

*"Добавь сохранение данных: когда я закрываю страницу и открываю снова, мои цифры должны остаться"*

Чат дает код с localStorage. Вставляем.

Не работает. Ошибка в консоли.

Пробуем объяснить чату проблему:

*"При обновлении страницы данные не загружаются. Вот ошибка в консоли: [копируем ошибку]"*

Чат дает новый код. Вставляем.

Работает, но частично. Одно поле сохраняется, другие — нет.

**Проблема:** Начинается путаница. Чат забывает, что было раньше. Код растет, становится сложнее отследить изменения.

---

### [08:00 - 09:30] ТРИ ОГРАНИЧЕНИЯ ЧАТА (Voice-over + слайды)

**ВИЗУАЛ:** Слайды с проблемами

**ТЕКСТ:**

Ты только что увидел магию и границу одновременно.

**Ограничение 1: "Золотая рыбка"**

Как только код вырос больше 200 строк, чат начал забывать структуру. Он дает решение для одной части, ломая другую.

**Когда проявляется:**
- Проект больше 1 файла
- Много функций, связанных друг с другом
- Нужно помнить, что было 5 сообщений назад

**Ограничение 2: "Копипаст-ад"**

Ты постоянно копируешь код туда-сюда:
- Чат → файл
- Файл → чат (когда объясняешь ошибку)
- Чат → файл (новая версия)

Это не работа — это транспортировка.

**Когда критично:**
- Более 5 итераций
- Нужно править несколько файлов
- Сложные изменения

**Ограничение 3: "Нет версий"**

Когда что-то сломалось, ты не можешь вернуться к рабочей версии.

Нет истории изменений. Нет кнопки "отменить последние 3 изменения".

**Когда больно:**
- Чат предложил "улучшение", которое все сломало
- Ты не помнишь, что было в рабочей версии
- Нужно начинать с нуля

---

### [09:30 - 10:30] ЧЕК-ЛИСТ: 3 СИГНАЛА ВЫЙТИ ИЗ ЧАТА (Слайд)

**ВИЗУАЛ:** Слайд с чек-листом

**ТЕКСТ:**

**ЧЕК-ЛИСТ: КОГДА ПОРА ПЕРЕХОДИТЬ НА СЛЕДУЮЩИЙ УРОВЕНЬ**

**Сигнал 1: Проект больше 3 файлов**
- Чат начинает путаться
- Изменение в одном файле ломает другой
- Ты больше времени тратишь на координацию, чем на создание

➡️ **Решение:** Bolt/Lovable (урок 6) или IDE (урок 9)

**Сигнал 2: Больше 10 итераций на одной задаче**
- Каждое исправление создает новую проблему
- Чат предлагает переписать всё с нуля
- Ты потерял контроль над кодом

➡️ **Решение:** IDE + агент с контекстом проекта (урок 9-12)

**Сигнал 3: Нужны версии и откаты**
- Боишься что-то менять (может сломаться)
- Уже было несколько "всё переписываем заново"
- Хочешь экспериментировать без страха

➡️ **Решение:** Git + GitHub (уроки 10-11)
