# Tech Context: Технологии и инструменты

## Технологический стек курса

### Что изучают студенты

#### 1. IDE (Visual Studio Code)
- **Назначение:** Единое рабочее пространство для всех файлов проекта
- **Простое объяснение:** «Умная кухня, где всё под рукой, а не разбросано по квартире»
- **Ключевые возможности:**
  - Просмотр всех файлов проекта
  - Встроенный терминал
  - Интеграция с Git
  - Расширения для работы с AI-агентами

#### 2. Git
- **Назначение:** Система контроля версий
- **Простое объяснение:** «Машина времени для проекта»
- **Ключевые возможности:**
  - История всех изменений
  - Сравнение версий (diff)
  - Ветки для экспериментов
  - Откат к любой точке

#### 3. GitHub
- **Назначение:** Облачное хранилище и платформа для совместной работы
- **Простое объяснение:** «Общий сейф в интернете»
- **Ключевые возможности:**
  - Резервное копирование
  - Доступ с любого устройства
  - Совместная работа
  - Портфолио проектов

#### 4. AI-Агенты
- **Основные инструменты:**
  - **Cline** — агент для VS Code
  - **Claude Code** — встроенный в Claude
- **Простое объяснение:** «Исполнитель с руками, который меняет файлы за вас»
- **Ключевые возможности:**
  - Видит всю структуру проекта
  - Читает и пишет файлы напрямую
  - Выполняет команды в терминале
  - Показывает diff перед применением

## Форматы данных

### Основной формат: Markdown (.md)
- **Почему:** Простой текст с минимальной разметкой
- **Применение:** 
  - Документация курса
  - Заметки студентов
  - Структура проектов
- **Преимущества:**
  - Читается как обычный текст
  - Легко конвертируется в HTML, PDF
  - Поддерживается всеми инструментами

### Дополнительные форматы
- **HTML** — для визуализации таблиц (Кейсы, Обоснования)
- **JSON/YAML** — для конфигураций (в будущем)

## Технические ограничения и решения

### Барьер входа для студентов
**Проблема:** Страх перед «программистскими» инструментами  
**Решение:**
- Пошаговая установка с скриншотами
- Простые объяснения без жаргона
- Аналогии из знакомых областей
- Акцент на результате, а не на технологии

### Кроссплатформенность
**Требование:** Курс должен работать на Windows, Mac, Linux  
**Решение:**
- VS Code — доступен на всех платформах
- Git — универсальный инструмент
- Онлайн-альтернативы (GitHub Codespaces) для тех, кто не хочет ставить ПО

### Ограничения AI-агентов
**Проблема:** Агенты могут ошибаться и «забывать» контекст  
**Решение:**
- Фреймворк проектирования (инструкции для агента)
- Memory Bank как внешняя память
- Git как страховка от ошибок
- Обучение студентов проверке и контролю
