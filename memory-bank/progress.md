## Эволюция решений

### Изменение 1: От блоков к урокам (31.01.2026)

**Было:**
- Структура: ЗАЧЕМ → ЧТО → КАК → ВСТРАИВАНИЕ (4 блока)
- Разделение мотивации и практики

**Стало:**
- 28 уроков без блоков
- Каждый урок = игровой цикл (мотивация + практика вместе)

**Почему:**
- Блочная структура создает разрыв между "зачем" и "как"
- В голове студента это не связывается
- Игровая механика удерживает вовлеченность

### Изменение 2: "Узнать" → "Получить" (31.01.2026)

**Было:**
- Формулировки: "вы узнаете", "вы поймете"
- Фокус на процессе обучения

**Стало:**
- Формулировки: "вы получите", "вы сможете"
- Фокус на результате и немедленном применении

**Почему:**
- Люди не хотят учиться — они хотят получать результаты
- Обучение = цена за результат, а не цель

### Изменение 3: Добавлены промежуточные инструменты (31.01.2026)

**Было:**
- Сразу к профессиональной среде (VS Code + Git + GitHub)

**Стало:**
- Прогрессия: чат → Bolt/Lovable → профессиональная среда

**Почему:**
- Нужно показывать рост возможностей
- Нельзя пугать сложностью сразу
- Промежуточные инструменты = плавный переход
