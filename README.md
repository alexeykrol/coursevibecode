# Проект: Курс «Вайб-кодинг и AI-агенты»

Этот репозиторий содержит материалы по разработке и проектированию образовательного курса, посвященного переходу от простого использования ИИ-чатов к профессиональной работе с автономными AI-агентами.

## 🎯 Суть проекта

Мы создаем методологию и программу обучения для профессионалов, которые хотят преодолеть ограничения обычных чат-ботов (ChatGPT, Claude в браузере) и освоить инженерный подход к работе с текстом, знаниями и кодом.

**Ключевая идея:** Переход от парадигмы «ИИ-советник» (чат) к парадигме «ИИ-сотрудник» (агент в IDE).

## 🛠 Техническое ядро (Стек)

Курс учит использовать «связку трех» для любых интеллектуальных задач:
1.  **IDE (VS Code):** Единое рабочее пространство (вместо разрозненных файлов).
2.  **Git / GitHub:** «Машина времени» и надежный сейф для проекта.
3.  **AI-Агенты (Cline / Claude Code):** Исполнители, которые имеют доступ к файловой системе и могут выполнять работу автономно.

## 👥 Для кого этот курс

*   **Аудитория:** Профессионалы 40–60 лет, эксперты в своих нишах (юристы, методологи, предприниматели, авторы).
*   **Проблема:** Уперлись в «стеклянный потолок» чат-ботов (потеря контекста, ручной копипаст, хаос в файлах).
*   **Потребность:** Нужна система, контроль, масштаб и возможность создавать собственные инструменты (приложения, автоматизации) без глубокого знания программирования.

## 🚀 Цели курса

1.  **Революция производительности:** Ускорение текущей работы (тексты, отчеты, анализ) в 10+ раз за счет доступа агента ко всем файлам проекта.
2.  **Революция возможностей:** Дать навык создания собственных веб-приложений и сервисов людям, которые никогда не писали код.
3.  **Суверенитет:** Переход от «аренды» контента на чужих платформах к созданию собственных цифровых активов, которые работают автономно.

## 📂 Структура репозитория

*   `Концепция программы курса.md` — общий каркас и принципы обучения.
*   `Программа курса.md` — детальная разбивка по блокам и урокам.
*   `Клиенты и проблемы.md` — анализ целевой аудитории и их болей (явных и латентных).
*   `Кейсы.html` / `Обоснования.html` — сравнительные таблицы «Было/Стало» для разных профессий.
*   `role.md` — описание ролевой модели взаимодействия (SME + Instructional Designer).
*   `AGENTS.md` — инструкции для AI-агентов по работе с этим проектом.

## 🔑 Статус проекта

Активная фаза проектирования архитектуры курса и проработки учебных материалов.
