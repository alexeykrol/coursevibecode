# Новая архитектура курса: Игровые циклы

**Статус:** 🟡 Черновик для обсуждения  
**Дата:** 31.01.2026  
**Концепция:** Курс как игра — последовательность маленьких циклов от простого к сложному

---

## Концепция игрового цикла

**Структура каждого цикла:**
1. **Возможность** — что ты сможешь делать (мотивация)
2. **Инструмент** — что для этого нужно
3. **Действие** — что сделать прямо сейчас
4. **Приз** — что ты получил/достиг

**Принцип:** Маленькие шаги, немедленные результаты, постоянное ощущение прогресса.

---

## Предлагаемая последовательность циклов

### 🎮 Цикл 1: Первая победа над чатом

**Возможность:**  
Ты сможешь работать с проектом из 100 файлов без копипаста. AI увидит весь контекст и сделает работу за тебя.

**Инструмент:**  
VS Code + AI-агент (Cline)

**Действие:**
1. Установи VS Code (3 мин)
2. Установи расширение Cline (2 мин)
3. Открой свой проект (или создай тестовый)
4. Дай первое задание агенту: "Найди все места, где упоминается X, и замени на Y"

**Приз:**  
Ты только что сделал за 2 минуты то, на что в чате ушло бы 2 часа копипаста. Ты увидел разницу своими глазами.

---

### 🎮 Цикл 2: Машина времени для твоих проектов

**Возможность:**  
Ты никогда больше не потеряешь ни одного изменения. Ты сможешь вернуться к любой версии за 10 секунд.

**Инструмент:**  
Git

**Действие:**
1. Установи Git (2 мин)
2. Инициализируй репозиторий в своем проекте
3. Сделай первый commit (сохрани текущее состояние)
4. Измени что-то в файле
5. Сделай второй commit
6. Откатись к первому commit — увидишь, как всё вернулось

**Приз:**  
Ты получил машину времени. Теперь можешь экспериментировать без страха что-то сломать. Всегда можешь вернуться назад.

---

### 🎮 Цикл 3: Облачный сейф для всей твоей жизни

**Возможность:**  
Твой компьютер может сгореть — твои проекты останутся в безопасности. Ты сможешь работать с любого устройства в любой точке мира.

**Инструмент:**  
GitHub

**Действие:**
1. Создай аккаунт на GitHub (3 мин)
2. Создай remote-репозиторий
3. Отправь свой проект в облако (git push)
4. Удали локальную копию проекта (!!)
5. Скачай её обратно из облака (git clone)

**Приз:**  
Ты только что доказал себе, что твои проекты в безопасности. Даже если компьютер сгорит — всё восстановится за минуту.

---

### 🎮 Цикл 4: Работа без копипаста на реальном проекте

**Возможность:**  
Ты возьмешь свой настоящий проект и решишь реальную задачу в 10 раз быстрее, чем раньше.

**Инструмент:**  
Вся связка: VS Code + Git + GitHub + AI-агент

**Действие:**
1. Открой свой реальный проект (курс, книга, документация, что угодно)
2. Сделай commit (зафиксируй текущее состояние)
3. Дай агенту задачу: "Проанализируй структуру и найди несоответствия"
4. Дай агенту задачу: "Измени форматирование всех заголовков"
5. Посмотри, что изменилось (git diff)
6. Сохрани результат (commit + push)

**Приз:**  
Ты решил реальную задачу из своей работы в 10 раз быстрее. Ты увидел конкретную экономию времени на своих цифрах.

---

### 🎮 Цикл 5: Создание первого автономного проекта

**Возможность:**  
Ты создашь что-то, что раньше требовало программиста: простое приложение, автоматизацию, систему.

**Инструмент:**  
Полная связка + промптинг + понимание структуры

**Действие:**
1. Придумай простой проект (калькулятор, landing page, автоматизация)
2. Попроси агента создать структуру проекта
3. Попроси агента написать код
4. Запусти проект
5. Попроси агента исправить ошибки
6. Сохрани всё в GitHub

**Приз:**  
Ты создал свой первый цифровой актив. Он работает. Ты можешь его показать. Это твоё портфолио.

---

### 🎮 Цикл 6: Суверенитет — создание актива, который можно продать

**Возможность:**  
Ты создашь актив, которым владеешь на 100%. Который работает 24/7. Который можно масштабировать. Который можно продать.

**Инструмент:**  
Всё + понимание монетизации + стратегия

**Действие:**
1. Выбери идею для актива (используй фреймворк из урока)
2. Создай MVP за выходные с помощью агента
3. Опубликуй на GitHub Pages / Vercel / другой платформе
4. Получи первых пользователей
5. Собери обратную связь
6. Улучши с помощью агента

**Приз:**  
Ты создал актив, который работает независимо от тебя. Это точка входа в новый класс жизни. Ты больше не только исполнитель — ты владелец.

---

### 🎮 Цикл 7-10: Углубление и специализация (опционально)

**Возможные направления:**
- Цикл 7: Совместная работа (ветки, pull requests, code review)
- Цикл 8: Продвинутый промптинг для агентов
- Цикл 9: Интеграция с API и внешними сервисами
- Цикл 10: Монетизация и продажа своих проектов

## Вопросы для обсуждения с SME:

1. **Последовательность циклов:** Согласны с порядком 1-6? Логичен ли прогресс?
2. **Размер циклов:** Не слишком ли большие/маленькие шаги?
3. **Баланс сложности:** Правильно ли распределена нагрузка?
4. **Первый цикл:** Можно ли начинать сразу с установки VS Code + Cline, или нужна вводная мотивация?
5. **Циклы 7-10:** Нужны ли они или достаточно 6 основных?
6. **Призы:** Достаточно ли конкретны и мотивирующи результаты каждого цикла?
7. **Длительность:** Сколько времени должен занимать каждый цикл? (урок = цикл?)

## Ключевые отличия от старой структуры:

**Было:**
- Блок 1: Вся мотивация (6 уроков)
- Блок 2: Все инструменты (5 уроков)
- Блок 3: Вся практика (4 урока)
- Блок 4: Специализация (3 урока)

**Стало:**
- Цикл 1: Мотивация + инструмент + действие + приз
- Цикл 2: Мотивация + инструмент + действие + приз
- Цикл 3: Мотивация + инструмент + действие + приз
- ...и так далее

**Преимущества:**
- ✅ Немедленный результат после каждого шага
- ✅ Постоянное ощущение прогресса
- ✅ Мотивация и инструмент связаны напрямую
- ✅ Нет разрыва между "зачем" и "как"
- ✅ Игровая механика удерживает внимание
