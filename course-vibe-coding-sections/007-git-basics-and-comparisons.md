### **Как это обычно делают без Git**

Ты работаешь и время от времени:

- нажимаешь Save
- делаешь копии:
    
    project_final, project_final2, project_final_really, project_old
    

Если что-то сломалось — ты **надеешься**, что нужная копия сохранилась.

### **Что делают CAD / Unreal**

Они делают то же самое, но **автоматически**:

- много автосейвов
- история действий
- откат внутри одного проекта

Но:

- это **один файл / одна сцена**
- если файл повредился — всё
- если нужно два разных варианта — начинаются копии файлов

### **Что делает Git**

Git **сам**:

- сохраняет версии всей папки проекта;
- помнит, **что именно изменилось**;
- позволяет сказать:
    
    «Вот версия для клиента»
    
    «Вот версия, где я экспериментировал»
    
    «Вот рабочая версия месяц назад»
    

И главное отличие:

**CAD / Unreal спасают от случайных ошибок.**

**Git спасает от хаоса версий.**

Простой итог в одной строке:

**Git — это аккуратный архив вариантов проекта, который не требует ручных копий.**

Уровень уверенности: **95%**.

- GitHub

Коротко и максимально по-человечески.

**GitHub — это место, где хранятся проекты с Git и где люди работают над ними вместе.**

Если совсем приземлённо:

**Git** — это как система аккуратных сохранений **на твоём компьютере**.

**GitHub** — это **онлайн-полка**, куда ты кладёшь эти сохранения, чтобы:

— не потерять,

— открыть с другого компьютера,

— показать другим,

— работать вместе.

Что GitHub даёт на практике:

1. **Резервную копию в интернете**
    
    Если ноут умер — проект живёт на GitHub.
    
2. **Совместную работу без бардака**
    
    Несколько человек могут работать над одним проектом, не перетирая друг друга.
    
3. **Публичную витрину**
    
    Можно показать проект миру:
    
    — код
    
    — книгу
    
    — тексты
    
    — документацию
    
    — идеи
    

Поэтому GitHub часто используют как **портфолио**.

1. **Контроль изменений между людьми**
    
    Видно:
    
    — кто что изменил
    
    — когда
    
    — зачем
    

Если коротко одной фразой:

**Git — это память.**

**GitHub — это общий сейф и доска объявлений.**

Для ориентира: GitHub — это сервис компании **GitHub**.

Уровень уверенности: **95%**.

Ок, максимально просто, на бытовых примерах.
